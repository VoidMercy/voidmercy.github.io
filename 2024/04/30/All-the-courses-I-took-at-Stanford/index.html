All the courses I took at Stanford

2024-04-30

This post will reflect upon all the courses I took for my MS EE degree at Stanford. I will give my personal opinion of the quality of the course, but mainly key ideas and takeaways from the course. I took the following courses, listed in the order I took them in:

  • CS 257: Introduction to Automated Reasoning
  • CS 248A: Computer Graphics: Rendering, Geometry, and Image Manipulation
  • CS 217: Hardware Accelerators for Machine Learning
  • EE 382A: Parallel Processors Beyond Multicore Processing
  • EE 282: Computer Systems Architecture

CS 257: Introduction to Automated Reasoning

This course is about the theory behind SAT and SMT solvers, which are widely applicable in formal verification. The main ideas in this course are algorithmic ideas which enable SAT solving, which is an NP-complete problem. The key algorithm for SAT solving is DPLL, which is a backtracking-based brute-force algorithm which assigns truth values to variables and checks for satisfiability. However, obviously, as you introduce more SAT variables to solve for, the problem gets harder to solve. The key idea behind the extension to DPLL(T) is that theory solvers only need to solve a conjunction of literals (A and B and C and â€¦). The speed of a theory is largely related to the efficiency of the theory solver. Common theory and theory solver pairs include: (bitvectors, bit-blasting), (real number math, linear programming), (integer math, integer linear programming).

CS 248A: Computer Graphics: Rendering, Geometry, and Image Manipulation

This course is an excellent course which teaches all the fundamentals of computer graphics. It teaches the techniques you need to know to be able to produce any scene you see in the real world digitally. First, this course teaches the graphics pipeline, and introduces triangle geometry, coordinate system transformations, textures and texture mapping, and the triangle rasterization pipeline. Then, it goes onto representing objects using meshes and mesh editing data structures (half-edge) and techniques/algorithms. Then, it teaches rendering using ray-tracing (the fundamental reflection equation). The key idea behind making ray-tracing computationally tractable is that we use Monte Carlo sampling instead of integrating over all light directions. Additionally, to get less noisy data, we bias Monte Carlo sampling towards locations we know where there is a light. However, oftentimes there is still noise, so some deep learning supersampling techniques can be applied as post-processing. Another key idea in ray-tracing is the BVH data structure used for ray-mesh intersections.

CS 217: Hardware Accelerators for Machine Learning

This was a poor course. The main idea is that memory is often the bottleneck since compute is cheap to add, but getting data to your compute is expensive. We need to tile the data such that each layer of tiling fits inside some cache level in order to achieve higher arithmetic intensity and not excessively load/store to/from the DRAM.

EE 382A: Parallel Processors Beyond Multicore Processing

This was a unique course, which used a simulator of a SIMD architecture chip, with an array of processing elements which passed data between each other based on single instructions. The language is called Kestrel. Essentially the course consisted of assembly programming assignments, where we had to program algorithms for these machines, such as longest substring matching and mandlebrot fractal generation. The key idea is thinking about where in the algorithm can parallel computations occur, and to be careful about control. Another key idea is that you can partition your program into basic blocks and assign a state to each processing element based on which basic block it should currently be executing. This way, you can handle divergent control flows between processing elements. However, you can still only execute one basic-block at a time since thereâ€™s only 1 instruction, so many divergent PEs would cause poor efficiency since not all PEs are actively executing at once.

EE 282: Computer Systems Architecture

This was a typical computer architecture course, which talked about OOO CPUs, caches, memory technologies (SRAM, DRAM, flash), vector processors, threads, GPUs, accelerators, virtual memory, virtualization. It covers a lot of content, which is great. The key idea in OoO CPUs is exploiting instruction level parallelism using register dependency analysis algorithms (register renaming). Caches are important to exploit spatial and temporal locality. Threads can be interleaved within a core to hide latencies.

CS 229: Machine Learning

This was an excellent course about the theoretical fundamentals of machine learning. It closely ties machine learning with statistics, and introduces all of the statistics ideas and theories before ML. The key idea behind many early models is that we use probabilistic models that we can analytically solve for using maximum likelihood (linear, logistic and generalized linear models). Another fundamental idea is that if we have the right features, then a linear classifier will suffice. SVMs and kernel methods generated complex function combinations of features, in hopes that this representation will have some useful features that can be linearly classified to fit the data well. Machine learning is fundamentally an model that can generalize to arbitrary functions in order to learn features that can be linearly classified. The key enabler is the SGD optimization technique to learn the function to generate good features. Also, regularization is important to prevent overfitting, but in general overfitting is hard to deal with and many successful techniques seem quite empirical to me. The CS 229 course notes are excellent and worth reviewing in detail.

EE 277: Bandit Learning: Behaviors and Application

I have a separate post on this, but the key ideas are:

  • Model the environment using a probabilistic model. Unknowns are given a prior distribution.
  • Choose action which balances exploration of different actions and exploitation of optimal actions.
  • Based on actions and observations, compute the posterior distribution.
  • When choosing an action, we should consider both the expected reward and the information the action reveals about the uncertainty in the environment using shannon entropy.

CS 248B: Fundamentals of Computer Graphics: Animation and Simulation

This was not a very valuable course. Essentially the idea is to simulate things using differential equations that approximate physics, time-stepped using numerical methods. A lot of modern simluation techniques are data driven that use machine learning to fit a model to data.

CS 224N: Natural Language Processing with Deep Learning

This was an excellent course about everything language models. It gives intuition behind the techniques used in current large language models.

  • The key idea behind word embeddings is that they embed word in high dimensional space, where generally related words are closer. There is some structure in the high dimensional space, that enables stuff like analogies. Initial techniques of getting word embeddings is based off of the idea that words are defined by the company they keep (their contexts).
  • Recurrent neural networks are initial ideas of sequence to sequence models, but suffer from long distance context and gradient loss, and difficulty to parallelize the computation.
  • Attention mechanism is a deep learning technique to solve long distance context sharing. The key idea is that every token generates a query, which queries every other token through a query key dot products for every query between every token. It turns out attention is all you need, and thatâ€™s what transformers are.
  • An encoder architecture compresses a representation to extract key relevant features.
  • Pre-training is critical to get a very good prior weights initialization which then makes fine-tuning very quick and effective. Really, pre-training is one of the most important breakthroughs, and it surprisingly continues to scale very well as models get bigger, leading to foundational models.
